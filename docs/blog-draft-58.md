# PyPI에 패키지를 올리며 알게 된 것들

> 시리즈: 몰랐던 것을 알게 되다 #1

자바 개발자입니다. MCP 서버를 만들었고, 비개발자도 쓸 수 있게 PyPI에 배포해야 했습니다. 그 과정에서 몰랐던 것들을 정리합니다.

---

## PyPI = Python의 Maven Central

**PyPI**(Python Package Index)는 Python 패키지의 공개 저장소입니다. 누구나 무료로 올리고, 무료로 받습니다.

| 생태계 | 저장소 | 한 줄 실행 도구 |
|--------|--------|----------------|
| Java | Maven Central | jbang |
| Node.js | npm | npx |
| Python | **PyPI** | **uvx** |

핵심: **코드를 올려두는 곳**이지, 서버를 대신 운영해주는 곳이 아닙니다.

---

## uvx, npx, jbang — 같은 문제, 다른 생태계

세 도구 모두 하는 일은 같습니다. **패키지를 설치 없이 받아서 바로 실행**.

- `npx create-react-app` — npm 패키지를 임시로 받아 실행
- `jbang hello@jbangdev` — Java 코드를 빌드 없이 스크립트처럼 실행
- `uvx slack-to-notion-mcp` — PyPI 패키지를 격리 환경에서 실행

시스템에 아무것도 남기지 않고, 종료하면 정리됩니다.

> jbang은 자바 개발자 사이에서도 인지도가 높지 않습니다. "Java에도 npx 같은 게 있었어?"라는 반응이 보통입니다. 그만큼 이 개념 자체가 생태계를 넘어 공유되기 어렵습니다.

---

## 배포자 vs 사용자

가장 헷갈렸던 부분입니다. "내 토큰으로 모두가 쓰는 건가?"

**아닙니다.** 앱스토어와 같습니다.

- **배포자** (나, 1명): PyPI 토큰 발급 → GitHub에 등록 → 태그 push → 자동 배포
- **사용자** (누구나): `uvx slack-to-notion-mcp` 한 줄 실행. 토큰도, 계정도 필요 없음

PyPI 토큰은 앱스토어 **개발자 계정**이고, uvx 실행은 앱 **설치 버튼**입니다.

---

## 몰랐던 것 → 알게 된 것

| 몰랐던 것 | 알게 된 것 |
|-----------|-----------|
| PyPI | Python의 Maven Central. 무료 공개 패키지 저장소 |
| uvx | npx의 Python 버전. 격리 환경에서 한 줄 실행 |
| jbang | Java의 npx. 빌드 없이 스크립트처럼 실행 |
| PyPI 토큰 | 배포자 인증용. 사용자는 불필요 |
| 배포 vs 사용 | 올리는 사람 1명이 토큰 쓰고, 쓰는 사람은 그냥 실행 |

---

이런 것들을 기록하는 이유는 간단합니다. 까먹기 때문입니다. 반년 뒤에 "PyPI가 뭐였더라"라고 다시 검색할 수 있습니다. 그때 이 글을 꺼내보면 됩니다.

생태계가 다르면 당연한 것도 당연하지 않습니다. 모르는 건 아직 만나지 않은 것입니다.
