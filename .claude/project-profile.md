# Project Profile

프로젝트별 특수성을 정의하여 `/spec`과 `/implement` 스킬의 동작을 조정합니다.

## 산출물 유형

| 구분 | 내용 |
|------|------|
| 주요 산출물 | Claude Code 플러그인 (스킬, 설정, 스크립트) |
| 기능 | Slack 메시지/스레드 분석 → Notion 정리 및 이슈 생성 |
| 테스트 방법 | 실제 Slack 채널 메시지로 플러그인 동작 검증 |

## /spec 컨텍스트

`/spec` 호출 시 아래 관점으로 명세를 작성합니다.

- **설계 대상**: 플러그인 스킬(SKILL.md), API 연동 로직, 데이터 변환 규칙
- **명세 항목**:
  - Slack API 연동 범위 (채널 메시지, 스레드, 리액션 등)
  - Notion 데이터베이스 매핑 구조
  - 메시지 분석/분류 기준
  - 에러 핸들링 및 재시도 정책
- **다이어그램**: 데이터 흐름 Mermaid로 표현

## /implement 컨텍스트

`/implement` 호출 시 아래 관점으로 구현합니다.

- **구현 대상**: Slack API 연동, 메시지 파싱, Notion API 연동, 플러그인 스킬
- **구현 순서**:
  1. Slack 메시지 수집 로직
  2. 메시지 분석 및 구조화
  3. Notion 페이지/이슈 생성
  4. 플러그인 스킬 통합
- **검증 기준**:
  - Slack 메시지가 정상적으로 수집되는지 확인
  - Notion에 올바른 형식으로 생성되는지 확인
  - 중복 처리 방지 동작 확인

## 제약사항

- Slack API 토큰 및 Notion API 키는 환경변수 또는 설정 파일로 관리 (Git 추적 금지)
- 개인 메시지(DM)는 수집 대상에서 제외
- API Rate Limit을 고려한 요청 설계
